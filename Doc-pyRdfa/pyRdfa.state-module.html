Module state

source code

Parser's execution context (a.k.a. state) object and handling. The state includes:

  • language, retrieved from @xml:lang or @lang
  • URI base, determined by <base> or set explicitly. This is a little bit superfluous, because the current RDFa syntax does not make use of @xml:base; i.e., this could be a global value. But the structure is prepared to add @xml:base easily, if needed.
  • options, in the form of an options instance
  • a separate vocabulary/CURIE handling resource, in the form of an termorcurie instance

The execution context object is also used to handle URI-s, CURIE-s, terms, etc.
