add_triples(self, msg, top_class, info_class, context, node)

source code 

Add an error structure to the processor graph: a bnode with a number of predicates. The structure follows the processor graph vocabulary as described on the RDFa WG Wiki page.

Parameters:
  • msg - the core error message, added as an object to a dc:description
  • top_class (URIRef) - Error, Warning, or Info; an explicit rdf:type added to the bnode
  • info_class (URIRef) - An additional error class, added as an rdf:type to the bnode in case it is not None
  • context (either an URIRef or a URI String (an URIRef will be created in the second case)) - An additional information added, if not None, as an object with rdfa:context as a predicate
  • node (string) - The node's element name that contains the error
Returns: BNode
the bnode that serves as a subject for the errors. The caller may add additional information

add_http_context(self, subj, http_code)

source code 

Add an additional HTTP context to a message with subject in subj, using the vocabulary. Typically used to extend an error structure, as created by add_triples.

Parameters:
  • subj - an RDFLib resource, typically a blank node
  • http_code - HTTP status code
